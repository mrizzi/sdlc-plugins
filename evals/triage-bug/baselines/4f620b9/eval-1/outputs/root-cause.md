## Root Cause

The plan-feature skill's convention extraction logic parses `CONVENTIONS.md` headings using
`line[3:]` to extract the section name after `## `. This slice does not strip trailing
whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `),
the extracted key becomes `"Migration Patterns  "` instead of `"Migration Patterns"`.

The subsequent convention matching step uses exact string comparison
(`convention_name in discovered_conventions`), which fails because the lookup key does not
have trailing whitespace while the stored key does. The convention is silently skipped with
no warning or error logged.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention extraction loop
  (`section_name = line[3:]` missing `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention matching logic
  (`convention_name in discovered_conventions` performs exact match against unstripped keys)

## Suggested Approach

1. Add `.strip()` to the heading extraction: `section_name = line[3:].strip()`
2. Consider also normalizing the lookup key for defensive matching
3. Add a warning/log when an expected convention section is referenced but not found in the
   parsed conventions dictionary

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on a heading line (e.g.,
`## Migration Patterns  ` with trailing spaces). Run the convention extraction and matching
logic against this fixture. Assert that the convention IS matched and included in the
generated task's Implementation Notes, despite the trailing whitespace.

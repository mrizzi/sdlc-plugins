## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md section
headings using `line[3:]`, which does not strip trailing whitespace. When a heading line
contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name
becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. This causes the
subsequent exact-match lookup in the convention-aware task enrichment to fail silently --
the convention is skipped without any warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- Convention conformance analysis section
  - Heading extraction: `section_name = line[3:]` (missing `.strip()`)
  - Convention-aware task enrichment: exact-match lookup `if convention_name in discovered_conventions`

## Suggested Approach

Add `.strip()` to the heading extraction line to normalize section names:

```
section_name = line[3:].strip()
```

This ensures trailing whitespace (spaces, tabs, newlines) is removed from heading names
before they are stored in the conventions dictionary, allowing exact-match lookups to
succeed regardless of whitespace in the source CONVENTIONS.md file.

## Reproducer Strategy

Create a test that:

1. Provides a CONVENTIONS.md fixture with trailing whitespace on a heading line
   (e.g., `## Migration Patterns  ` with trailing spaces)
2. Runs the convention extraction logic against this fixture
3. Asserts that the convention is found when looked up by the clean name
   `"Migration Patterns"` (without trailing spaces)
4. Asserts that the generated task's Implementation Notes include the expected
   convention reference: `Per CONVENTIONS.md §Migration Patterns: ...`

The test should fail before the fix (demonstrating the bug) and pass after the fix.

## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` section
headings using `line[3:]`, which takes the text after `## ` but does not strip trailing
whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `),
the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The
downstream task enrichment step performs an exact-match lookup against canonical convention
names (without trailing spaces), so the match fails silently and the convention is omitted
from the generated task description.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention conformance analysis
  section, specifically the heading extraction logic: `section_name = line[3:]`
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment
  section, specifically the match: `if convention_name in discovered_conventions`

## Suggested Approach

Apply `.strip()` (or `.rstrip()`) to the heading extraction to normalize trailing whitespace:

```python
section_name = line[3:].strip()
```

This ensures that headings with trailing whitespace are treated identically to headings
without it. Additionally, consider adding a warning log when a convention heading is
discovered but not matched during task enrichment, to make similar issues visible in
the future.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on one or more section
headings (e.g., `## Migration Patterns  ` with trailing spaces). Run the plan-feature
convention conformance analysis against this fixture and assert that the convention is
correctly matched and included in the generated task's Implementation Notes. The test
should fail before the fix (convention silently dropped) and pass after the fix
(convention correctly included).

## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md`
section headings using `line[3:]`, which does not strip trailing whitespace. When a
heading line contains trailing spaces (e.g., `## Migration Patterns  `), the
extracted section name becomes `"Migration Patterns  "` (with trailing spaces).
The subsequent convention-matching step performs an exact-match lookup using the
clean name `"Migration Patterns"`, which fails because the dictionary key includes
the trailing spaces. The convention is silently dropped from the generated task
description with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention heading
  extraction logic (`line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task
  enrichment logic (exact-match lookup against extracted section names)

## Suggested Approach

Apply `.strip()` to the heading extraction to normalize trailing whitespace:

Change `section_name = line[3:]` to `section_name = line[3:].strip()`.

This ensures that heading names in the conventions dictionary are always clean,
regardless of whitespace in the source file. No changes are needed to the
convention-matching step, which already uses clean names.

## Reproducer Strategy

1. Create a `CONVENTIONS.md` fixture with trailing whitespace on a heading line
   (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention extraction logic against this fixture.
3. Assert that the extracted section name is `"Migration Patterns"` (without
   trailing whitespace).
4. Assert that the convention-matching step successfully includes the convention
   in the generated task's Implementation Notes.

The reproducer test should fail before the fix (section name includes trailing
spaces, convention is dropped) and pass after the fix (section name is stripped,
convention is included).

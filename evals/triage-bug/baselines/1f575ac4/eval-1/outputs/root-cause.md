## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md section headings using `line[3:]`, which captures everything after the `## ` prefix without stripping trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent exact-match lookup in the task enrichment step fails silently because the dictionary key includes the trailing whitespace while the lookup key does not. No warning is logged, so the user has no indication that a convention was skipped.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention conformance analysis section, heading extraction logic (`line[3:]`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment section, exact-match lookup (`if convention_name in discovered_conventions`)

## Suggested Approach

Apply `.strip()` to the heading extraction to normalize whitespace:
```python
section_name = line[3:].strip()
```
This ensures trailing (and leading) whitespace on CONVENTIONS.md headings is removed before storing section names in the conventions dictionary. The exact-match lookup will then succeed regardless of whitespace formatting in the source file.

## Reproducer Strategy

1. Create a CONVENTIONS.md fixture with trailing whitespace on a heading (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention parsing logic against this fixture.
3. Assert that the convention is correctly extracted with the section name `"Migration Patterns"` (without trailing whitespace).
4. Assert that the generated task's Implementation Notes include the expected convention reference: `Per CONVENTIONS.md Migration Patterns: add Index::create() for all FK columns.`

## Root Cause

The plan-feature skill's convention heading extraction uses `line[3:]` to extract the section name
after the `## ` prefix, but does not strip trailing whitespace. When a `CONVENTIONS.md` heading
line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted key becomes
`"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream convention-to-task
matching performs an exact string comparison against the clean (unpadded) convention name, which
fails silently -- no warning or error is emitted, and the convention is simply omitted from the
generated task description.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction loop
  (`section_name = line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment
  (exact-match lookup `if convention_name in discovered_conventions`)

## Suggested Approach

1. Add `.strip()` to the heading extraction: change `section_name = line[3:]` to
   `section_name = line[3:].strip()` so that trailing whitespace is removed before storing the
   section name in the conventions dictionary.
2. Consider adding a warning log when a convention name is expected but not found in the
   discovered conventions, to prevent silent failures in the future.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on at least one heading line
(e.g., `## Migration Patterns  ` with trailing spaces). Run the plan-feature convention
conformance analysis against this fixture and assert that:
- The convention IS matched despite trailing whitespace on the heading.
- The generated task's Implementation Notes include the expected convention reference
  (e.g., `Per CONVENTIONS.md Migration Patterns: ...`).

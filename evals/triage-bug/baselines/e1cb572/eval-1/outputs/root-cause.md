## Root Cause

The plan-feature skill's convention extraction logic in `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` extracts CONVENTIONS.md section headings using `line[3:]` without stripping trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted key becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream convention matching performs an exact string lookup, which fails because the padded key does not match the expected clean name. The convention is silently dropped with no warning or error logged.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention heading extraction (`line[3:]` missing `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment (exact match against padded key)

## Suggested Approach

Add `.strip()` to the heading extraction to normalize trailing whitespace: change `line[3:]` to `line[3:].strip()`. This ensures section names are stored without trailing whitespace regardless of the CONVENTIONS.md formatting. The downstream exact-match logic then works correctly without modification.

## Root Cause

The plan-feature skill's convention lookup extracts CONVENTIONS.md section headings
using `line[3:]`, which captures all characters after the `## ` prefix but does not
strip trailing whitespace. When a heading line contains trailing spaces (e.g.,
`## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "`
instead of `"Migration Patterns"`. The subsequent convention-aware task enrichment step
performs exact string matching (`if convention_name in discovered_conventions`), which
fails because the key with trailing spaces does not match the expected clean key.
The convention is silently dropped with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention lookup loop
  (`section_name = line[3:]`) and convention-aware task enrichment
  (`if convention_name in discovered_conventions`)

## Suggested Approach

Add `.strip()` to the heading extraction to normalize whitespace:
`section_name = line[3:].strip()`. This ensures trailing whitespace on heading lines
does not prevent convention matching. Additionally, consider emitting a warning when
a heading contains trailing whitespace to aid debugging.

## Reproducer Strategy

1. Create a CONVENTIONS.md fixture with trailing whitespace on at least one heading
   (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention lookup logic against this fixture.
3. Assert that the convention section is correctly extracted with a clean key
   (no trailing whitespace).
4. Assert that the convention is matched and included in the generated task's
   Implementation Notes.

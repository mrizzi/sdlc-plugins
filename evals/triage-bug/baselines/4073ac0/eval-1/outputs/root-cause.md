## Root Cause

The plan-feature skill's convention lookup extracts `CONVENTIONS.md` headings using `line[3:]`, which captures all characters after the `## ` prefix — including any trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The subsequent exact-match comparison against the expected convention name `"Migration Patterns"` fails, and the convention is silently dropped with no warning.

**What is broken**: The heading extraction in plan-feature's convention conformance analysis does not strip trailing whitespace from extracted heading names.

**Where it is broken**: `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — the convention conformance analysis section, specifically the line `section_name = line[3:]`.

**Why it is broken**: The `line[3:]` slice preserves all trailing characters (spaces, tabs) from the heading line. No `.strip()` or `.rstrip()` call is applied before storing the section name in the `conventions` dictionary.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis (heading extraction loop)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment (exact-match lookup)

## Suggested Approach

Apply `.strip()` to the extracted heading text so that trailing whitespace is removed before the section name is stored:

```
section_name = line[3:].strip()
```

This ensures the convention name matches regardless of trailing whitespace in the `CONVENTIONS.md` file.

How to verify the fix:

2. Run plan-feature against a feature that should match that convention.

4. The reproducer test should fail before the fix (convention silently dropped) and pass after the fix (convention correctly matched and included).

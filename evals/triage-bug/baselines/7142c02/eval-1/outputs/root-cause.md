## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md`
section headings using `line[3:]`, which takes the substring after the `## ` prefix
but does not strip trailing whitespace. When a heading line contains trailing spaces
(e.g., `## Migration Patterns  `), the extracted section name retains those spaces
(`"Migration Patterns  "`). The subsequent convention-aware task enrichment step
uses an exact-match lookup (`convention_name in discovered_conventions`), which fails
because the expected key `"Migration Patterns"` does not match the stored key
`"Migration Patterns  "`. The mismatch causes the convention to be silently dropped
from the generated task -- no warning or error is produced.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention conformance
  analysis section, specifically:
  - Heading extraction: `section_name = line[3:]` (missing `.strip()`)
  - Convention matching: `if convention_name in discovered_conventions`

## Suggested Approach

Apply `.strip()` to the extracted section name so that trailing (and leading)
whitespace is removed before storing it in the conventions dictionary:
```python
section_name = line[3:].strip()
```

Additionally, consider adding a warning log when a convention section is discovered
in `CONVENTIONS.md` but not matched during task enrichment, to prevent silent
failures in the future.

## Reproducer Strategy

Create a test with a `CONVENTIONS.md` fixture that includes trailing whitespace on
a section heading (e.g., `## Migration Patterns  ` with two trailing spaces). Run
the plan-feature convention extraction logic and verify that the convention is
correctly matched and included in the generated task's Implementation Notes. The
test should:

1. Assert that the convention IS matched despite trailing whitespace (fails before
   fix, passes after fix).
2. Assert that the generated task's Implementation Notes contain the expected
   convention reference (e.g., `Per CONVENTIONS.md Migration Patterns: ...`).

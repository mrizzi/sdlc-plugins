## Architecture Summary

This feature adds a side-by-side SBOM comparison capability across two repositories:

**Backend (trustify-backend):** A new comparison service computes structured diffs between two SBOMs on-the-fly using existing package, advisory, and license data (no new database tables). The service is exposed via a new `GET /api/v2/sbom/compare?left={id}&right={id}` endpoint that returns six diff categories: added packages, removed packages, version changes, new vulnerabilities, resolved vulnerabilities, and license changes.

**Frontend (trustify-ui):** A new comparison page at `/sbom/compare` renders the diff results using PatternFly components (ExpandableSection for diff categories, Table for data, Badge for counts, EmptyState for initial state). Users initiate comparisons from the SBOM list page by selecting two SBOMs with checkboxes and clicking "Compare selected". The comparison URL is shareable via query parameters.

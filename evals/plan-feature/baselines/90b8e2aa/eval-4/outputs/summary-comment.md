## Architecture Summary

The license compliance report is implemented as an extension of the existing SBOM module,
following the established `model/ + service/ + endpoints/` pattern:

- **Models** (Task 1): `LicenseReportSummary` and `LicenseGroup` structs define the
  API response shape. `LicensePolicy` struct loads a JSON configuration file that
  defines allowed/denied license identifiers for compliance checking.
- **Service + Endpoint** (Task 2): A new `license_report` method on `SbomService`
  queries `package_license` and `sbom_package` entities to aggregate packages by
  license, walks the transitive dependency tree, and checks compliance against the
  policy. The `GET /api/v2/sbom/{id}/license-report` endpoint exposes this via Axum.
- **Tests** (Task 3): Integration tests in `tests/api/license_report.rs` verify
  correct grouping, compliance flagging, transitive dependency inclusion, and edge
  cases against a real PostgreSQL test database.

No new database tables or migrations are introduced — the feature aggregates from
existing `package_license` and `sbom_package` entities.

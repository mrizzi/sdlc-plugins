## Architecture Summary

The feature adds a new server-side aggregation endpoint `GET /api/v2/sbom/{id}/advisory-summary` to the trustify-backend service. The implementation follows the established module pattern (model/ + service/ + endpoints/):

1. **Data layer** (Task 1): A new `AdvisorySeveritySummary` response model and a `SbomService` method that queries the `sbom_advisory` join table to aggregate unique advisory counts grouped by severity.
2. **API layer** (Task 2): An Axum endpoint handler registered under the existing SBOM route group, with 5-minute cache via tower-http and an optional `?threshold` query parameter for severity filtering.
3. **Cache coherence** (Task 3): Cache invalidation hook in the advisory ingestion pipeline to ensure summaries are refreshed when new advisories are linked to an SBOM.
4. **Quality assurance** (Task 4): Integration tests covering happy path, 404, deduplication, threshold filtering, and cache behavior, using the established real PostgreSQL test database pattern.

No new database tables are required -- the implementation uses the existing `sbom_advisory` join table and `advisory` entity.

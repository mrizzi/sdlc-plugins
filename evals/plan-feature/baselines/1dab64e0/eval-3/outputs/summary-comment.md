## Architecture Summary

The feature adds a side-by-side SBOM comparison capability across two repositories:

**Backend (trustify-backend):** A new `SbomCompareService` computes structured diffs between two SBOMs by comparing their package sets, advisory associations, and license values. The diff result is exposed via `GET /api/v2/sbom/compare?left={id1}&right={id2}`, returning six diff categories (added/removed packages, version changes, new/resolved vulnerabilities, license changes). No new database tables are required — diffs are computed on-the-fly from existing entity data.

**Frontend (trustify-ui):** A new comparison page at `/sbom/compare` implements the Figma design using PatternFly 5 components: `Select` dropdowns for SBOM selection, `ExpandableSection` with colored `Badge` counts for each diff category, composable `Table` for diff data, and `EmptyState`/`Skeleton` for empty and loading states. The existing `SeverityBadge` component is reused for vulnerability severity display. The SBOM list page gains checkbox selection and a "Compare selected" button for the comparison workflow entry point. URL query parameters enable shareable comparisons.

**Workflow mode:** `feature-branch` — the frontend comparison page depends on the backend endpoint that does not yet exist, requiring coordinated delivery. The `workflow:feature-branch` label will be applied to TC-9003.

## Workflow Mode Decision

**Selected mode**: `feature-branch`

**Rationale**: The following atomicity indicators were identified:

1. **Tightly coupled feature components** (indicator #4): The SBOM comparison UI (`/sbom/compare` page) requires the new backend endpoint `GET /api/v2/sbom/compare` that does not yet exist. 2. **Breaking API changes** (indicator #2): The frontend API types, client function, and React Query hook are designed around the specific response shape of the new comparison endpoint. If the backend endpoint is merged to main independently and its response shape changes before the frontend is merged, the frontend code would break.

**Interdependent tasks**:
- Frontend Task 4 (API types and hook) depends on Backend Task 3 (comparison endpoint) — the TypeScript interfaces must match the Rust response structs
- Frontend Task 5 (comparison page) depends on Task 4 (hook) and transitively on the backend endpoint
- Frontend Task 6 (list page compare action) depends on Task 5 (comparison page must exist for navigation)

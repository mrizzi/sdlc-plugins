## Architecture Summary

The feature adds a vulnerability remediation tracking dashboard with a backend-first, API-driven architecture:

1. **Backend (trustify-backend):** A new `remediation` domain module follows the established model/service/endpoints pattern. The `RemediationService` computes aggregations from existing entity tables (advisory, sbom_advisory, sbom) using SQL GROUP BY queries — no new database tables are introduced per the non-functional requirements. Two new REST endpoints expose the aggregated data: `GET /api/v2/remediation/summary` for severity x status counts and `GET /api/v2/remediation/by-product` for paginated per-product breakdowns.

2. **Frontend (trustify-ui):** A new `RemediationDashboardPage` at `/remediation` consumes the backend endpoints via typed API client functions and React Query hooks. The page displays summary cards (Open/In Progress/Resolved totals with severity breakdown), a progress chart (30-day remediation trend), and a filterable vulnerability table (filter by severity, product, and status with server-side pagination for up to 10,000 entries).

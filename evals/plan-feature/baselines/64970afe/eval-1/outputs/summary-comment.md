### Architecture Summary

The implementation adds a new server-side aggregation endpoint to eliminate client-side advisory severity counting. The changes follow the established `model/ + service/ + endpoints/` module pattern within the `sbom` domain module of `modules/fundamental/`. A new `SeveritySummary` model struct is introduced, backed by a service method that queries the existing `sbom_advisory` join table with `SELECT DISTINCT` deduplication and groups by severity level. The endpoint uses `tower-http` caching middleware with a 5-minute TTL, and cache invalidation is integrated into the advisory ingestion pipeline (`modules/ingestor/`). An optional `?threshold` query parameter supports alerting integrations. No new database tables or migrations are required — the feature uses existing `sbom_advisory` and `advisory` entities.

## Architecture Summary

The backend adds a new `remediation` domain module under `modules/fundamental/src/` following the established model/service/endpoints pattern used by the advisory and sbom modules. Two new REST endpoints provide aggregated remediation data computed from existing vulnerability and SBOM relationship tables (no new database tables or migrations): `GET /api/v2/remediation/summary` returns severity-by-status counts, and `GET /api/v2/remediation/by-product` returns per-product breakdowns with pagination support.

The frontend adds a new dashboard page at `/remediation` built with PatternFly 5 components. The data layer follows the established pattern: typed API client functions in `src/api/rest.ts`, TypeScript interfaces in `src/api/models.ts`, and React Query hooks in `src/hooks/`. The dashboard includes summary cards (Open/In Progress/Resolved counts), a 30-day remediation trend chart, and a filterable vulnerability table supporting severity, product, and status filters with performance handling for up to 10,000 entries.

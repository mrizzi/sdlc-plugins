## Architecture Summary

This feature replaces the `advisory_status` lookup table with a PostgreSQL enum column (`advisory_status_enum`) directly on the `advisory` table. The change eliminates a join on every advisory query, simplifying the data model and improving query performance (target: ~40ms p95 latency reduction on the advisory list endpoint). The migration is atomic and reversible -- all five schema operations (create enum type, add column, backfill, drop FK, drop table) execute within a single transaction. All advisory-related code paths -- service layer, HTTP endpoints, ingestion pipeline, and integration tests -- are updated to use the enum column directly.

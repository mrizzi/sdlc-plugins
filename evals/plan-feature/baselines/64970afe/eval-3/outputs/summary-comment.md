## Architecture Summary

The feature adds a side-by-side SBOM comparison capability spanning both repositories:

**Backend (trustify-backend):** A new comparison module under `modules/fundamental/src/sbom/` follows the existing model/ + service/ + endpoints/ pattern. The service computes structured diffs in-memory from existing package, advisory, and license entity data (no new database tables). The endpoint at `GET /api/v2/sbom/compare?left={id1}&right={id2}` returns six diff categories: added packages, removed packages, version changes, new vulnerabilities, resolved vulnerabilities, and license changes.

**Frontend (trustify-ui):** A new comparison page at `/sbom/compare` follows the Figma design using PatternFly 5 components (Select dropdowns, ExpandableSection for diff sections, Badge for counts, composable Table for data, EmptyState for initial view, Skeleton for loading). The page reads SBOM IDs from URL query params for shareability. The SBOM list page gains checkbox selection and a "Compare selected" button for triggering comparisons.

**Workflow mode:** `feature-branch` — the frontend and backend are tightly coupled (the comparison page requires the comparison endpoint). All intermediate tasks target the `TC-9003` feature branch. The `workflow:feature-branch` label will be applied to the feature issue.

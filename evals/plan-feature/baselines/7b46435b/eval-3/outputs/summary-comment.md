## Architecture Summary

The feature adds a new SBOM comparison capability spanning both the backend API and frontend UI:

1. **Backend Model** (`SbomComparisonResult` struct and sub-structs) -- defines the response shape with six diff categories: added/removed packages, version changes, new/resolved vulnerabilities, and license changes
2. **Backend Service** (`SbomService::compare_sboms`) -- computes the diff on-the-fly by querying existing `sbom_package`, `package`, `package_license`, `sbom_advisory`, and `advisory` entities using SeaORM joins and set difference operations. No new database tables are required.
3. **Backend Endpoint** (`GET /api/v2/sbom/compare?left={id1}&right={id2}`) -- Axum handler accepting two SBOM UUIDs as query parameters, returning the structured diff as JSON with 400/404 error handling
4. **Frontend API Layer** -- TypeScript interfaces matching the backend response shape, Axios client function, and React Query hook (`useSbomCompare`) with conditional query enablement
5. **Frontend Comparison Page** (`/sbom/compare`) -- PatternFly-based UI with SBOM Select dropdowns, six ExpandableSection diff panels with Tables, EmptyState, Skeleton loading, and URL shareability via query parameters
6. **Frontend List Page Enhancement** -- checkbox selection and "Compare selected" navigation action on the existing SBOM list page

Cross-repo dependency ordering: backend tasks (1-3) must be completed before frontend tasks (4-6) that depend on the API contract. Task 4 explicitly depends on Task 2.

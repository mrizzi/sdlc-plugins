## Architecture Summary

The feature adds a new `GET /api/v2/sbom/{id}/license-report` endpoint to the existing SBOM module following the established `model/ + service/ + endpoints/` pattern. The implementation layers are:

1. **Model** (`LicenseComplianceReport`, `LicenseGroup`, `PackageLicenseEntry`, `LicensePolicy` structs) -- defines the response shape with packages grouped by license and compliance flags, plus configurable license policy
2. **Service** (`SbomService::license_compliance_report`) -- queries `sbom_package` joined with `package` and `package_license` entities, walks transitive dependencies, groups by license, and applies policy-based compliance flagging
3. **Endpoint** -- Axum handler returning 200 with the report or 404 for missing SBOMs
4. **Policy Configuration** -- JSON config file (`config/license-policy.json`) defining allowed/denied license lists, customizable per organization

No new database tables are required; the feature aggregates from existing `sbom_package`, `package`, and `package_license` entities.

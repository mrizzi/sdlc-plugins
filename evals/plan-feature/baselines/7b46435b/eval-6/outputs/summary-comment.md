## Architecture Summary

The feature adds a vulnerability remediation tracking dashboard with backend aggregation and a frontend dashboard page.

**Backend (trustify-backend):**
A new `remediation` domain module under `modules/fundamental/src/remediation/` following the established `model/ + service/ + endpoints/` pattern. Two REST endpoints are added:
1. `GET /api/v2/remediation/summary` -- returns aggregated severity x status counts (`RemediationSummary`)
2. `GET /api/v2/remediation/by-product` -- returns paginated per-product breakdown (`PaginatedResults<ProductRemediation>`)

No new database tables are created; all aggregations are computed from existing entity relationships (`sbom_advisory`, `advisory`, `sbom`, `package` entities). Caching is applied to the summary endpoint for p95 < 500ms performance.

**Frontend (trustify-ui):**
A new `RemediationDashboardPage` at `/remediation` using PatternFly 5 components with:
- Summary cards (Open, In Progress, Resolved counts)
- Progress chart (30-day trend)
- Filterable vulnerability table (severity, product, status filters)

The API layer adds TypeScript interfaces, Axios client functions, and React Query hooks following established patterns.

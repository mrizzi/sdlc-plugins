## Architecture Summary

The feature improves the existing `GET /api/v2/search` endpoint in the search module following the established `model/ + service/ + endpoints/` pattern. The implementation layers are:

1. **Migration** -- adds PostgreSQL GIN indexes on `tsvector` generated columns for SBOM, advisory, and package entities to enable fast full-text search
2. **Service** (`SearchService` in `modules/search/src/service/mod.rs`) -- modified to use `tsquery`/`ts_rank` for relevance scoring and to accept filter parameters for entity type and severity
3. **Endpoint** (`GET /api/v2/search` in `modules/search/src/endpoints/mod.rs`) -- extended with optional `entity_type` and `severity` query parameters; response includes new `relevance_score` field; all changes are additive and backwards-compatible
4. **Tests** (`tests/api/search.rs`) -- extended with comprehensive integration tests for relevance ordering, filters, combined filters, error handling, and backwards compatibility

No new database tables are required; the migration adds indexes and generated columns to existing tables. The existing `common/src/db/query.rs` filtering helpers are reused for filter composition.

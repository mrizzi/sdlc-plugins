## Architecture Summary

The feature adds a new `GET /api/v2/sbom/{id}/advisory-summary` endpoint to the existing SBOM module in the `trustify-backend` service. The implementation follows the established module pattern (`model/ + service/ + endpoints/`) and reuses existing infrastructure:

- **Model layer**: A new `AdvisorySeveritySummary` struct in `modules/fundamental/src/sbom/model/` provides the response shape with severity count fields.
- **Service layer**: `SbomService` gains an `advisory_severity_summary` method that queries the existing `sbom_advisory` join table and `advisory` entity to count unique advisories grouped by severity.
- **Endpoint layer**: A new handler in `modules/fundamental/src/sbom/endpoints/` registers the route, applies 5-minute caching via `tower-http`, and delegates to the service method.
- **Cache invalidation**: The advisory ingestion pipeline (`modules/ingestor/src/graph/advisory/`) is extended to invalidate cached summaries when new advisories are correlated with an SBOM.
- **Threshold filtering**: An optional `?threshold` query parameter enables alerting integrations to filter counts to only severities at or above a given level.

No new database tables are required — the implementation uses existing `sbom_advisory` relationship tables.

## Architecture Summary

The license compliance report feature is implemented as a new capability within the existing SBOM domain module, following the established `model/ + service/ + endpoints/` pattern:

1. **License Policy (common/)** — A shared `LicensePolicy` configuration model in `common/src/license_policy.rs` that loads allowed/denied license lists from a JSON config file and provides a compliance evaluation method.

2. **License Report Service (modules/fundamental/src/sbom/service/)** — A `LicenseReportService` that queries existing `sbom_package` and `package_license` entities to aggregate all packages for an SBOM, walks transitive dependencies, groups them by license type, and evaluates each group against the license policy.

3. **License Report Endpoint (modules/fundamental/src/sbom/endpoints/)** — A new `GET /api/v2/sbom/{id}/license-report` handler registered alongside existing SBOM routes, returning a JSON response with license groups, compliance flags, and summary counts.

4. **Integration Tests (tests/api/)** — Comprehensive end-to-end tests covering compliant SBOMs, non-compliant SBOMs, transitive dependency resolution, empty SBOMs, 404 handling, and CI/CD gate scenarios.

No new database tables are required — all data is aggregated from existing entities.

### Verdict: PASS

### Reasoning

The PR implements SPDX license validation with appropriate error handling:

1. **Validation logic** (`modules/fundamental/src/package/endpoints/list.rs`): The `validate_license_param` function iterates over each license identifier after splitting on commas. For each identifier, it calls `Expression::parse(id)`. If parsing fails (the identifier is not a valid SPDX expression), the function returns an `AppError::BadRequest` with a descriptive error message: `format!("Invalid SPDX license identifier: {}", id)`.

2. **Error propagation**: The `list_packages` handler calls `validate_license_param(license)?` using the `?` operator. If validation fails, the `AppError::BadRequest` is propagated as the handler's error response. Based on the repository's error handling conventions (the `AppError` enum in `common/src/error.rs` implements `IntoResponse`), this produces an HTTP 400 Bad Request response with the error message in the body.

3. **SPDX validation**: The `spdx::Expression::parse` function from the `spdx` crate validates against known SPDX license identifiers. `INVALID-999` is not a recognized SPDX license identifier, so parsing will fail and trigger the 400 response.

4. **Test coverage** (`tests/api/package.rs`): The `test_list_packages_invalid_license_returns_400` test requests `?license=INVALID-999` and asserts:
   - Response status is `StatusCode::BAD_REQUEST` (400)

   This directly validates the criterion. The error message is generated by the `format!` macro and included in the `AppError::BadRequest` variant.
